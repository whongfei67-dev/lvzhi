## 前置条件

### 1. 确保 API 服务运行

测试脚本需要 API 服务运行才能执行。

**方式一：使用 Docker**
```bash
cd /Users/wanghongfei/Desktop/智能体项目/律植项目

# 启动所有服务
docker compose up -d

# 或仅启动 API 服务
docker compose up -d api
```

**方式二：直接运行**
```bash
cd /Users/wanghongfei/Desktop/智能体项目/律植项目/apps/api

# 安装依赖（如果尚未安装）
pnpm install

# 启动 API 服务
pnpm dev
```

### 2. 验证服务状态

```bash
# 检查 API 是否运行
curl http://localhost:3001/health

# 预期响应
{"status":"ok","timestamp":"2026-04-02T00:00:00.000Z","version":"1.0.0"}
```

## 完整测试执行顺序

建议按以下顺序执行测试：

```bash
# 1. 健康检查（确认服务正常运行）
pnpm health-check

# 2. 部署检查清单（确认环境配置正确）
pnpm deploy:check

# 3. 数据库测试（确认数据库功能正常）
pnpm test:db:regression
pnpm test:db:consistency
pnpm audit:security

# 4. 功能流程测试（验证核心功能）
pnpm test:flows

# 5. 安全验证测试（确保安全性）
pnpm test:security:verify

# 6. 管理员测试（验证管理功能）
pnpm test:admin

# 7. E2E 测试（端到端验证）
pnpm test:e2e

# 8. 支付测试（验证支付功能）
pnpm test:payment
```

## 测试环境变量

测试脚本支持以下环境变量：

```bash
# API 基础地址（默认：http://localhost:3001）
export API_BASE_URL=http://localhost:3001

# 数据库连接
export DATABASE_URL=postgresql://user:pass@localhost:5432/lvzhi

export ZHIPU_API_KEY=your_key

# 通知渠道
export SLACK_WEBHOOK_URL=https://hooks.slack.com/...
export DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/...
```

## 常见问题

### Q: 脚本报错 "tsx not found"

**解决方案：**
```bash
# 安装 tsx
pnpm add tsx -D -r

# 或使用 npx
npx tsx scripts/health-check.ts
```

### Q: API 连接失败

**检查：**
```bash
# 确认 API 服务运行
curl http://localhost:3001/health

# 如果未运行，启动服务
docker compose up -d api
```

### Q: 数据库连接失败

**检查：**
```bash
# 确认数据库环境变量配置
cat deploy/.env | grep POSTGRES

# 确认数据库服务运行
docker compose ps postgres
```

## 测试报告

测试脚本会在终端输出彩色格式的结果：

| 状态 | 颜色 | 含义 |
|------|------|------|
| ✓ PASS | 绿色 | 测试通过 |
| ✗ FAIL | 红色 | 测试失败 |
| ⚠ WARN | 黄色 | 警告（可能有问题） |
| ℹ INFO | 蓝色 | 信息 |
