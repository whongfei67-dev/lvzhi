# 律植 - 压测准备指南

> 创建时间：2026-04-02
> 项目：律植法律智能体平台

---

## 压测概述

本指南帮助你准备和执行律植平台的性能压测。

### 压测目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| QPS (API) | ≥ 500 | 普通 API 接口吞吐量 |
| QPS (AI 对话) | ≥ 50 | AI 接口吞吐量 |
| P95 延迟 | ≤ 500ms | 95% 请求的响应时间 |
| P99 延迟 | ≤ 2s | 99% 请求的响应时间 |
| 错误率 | ≤ 1% | 失败请求比例 |
| CPU 使用率 | ≤ 80% | 服务器负载 |
| 内存使用率 | ≤ 85% | 内存使用情况 |

## 压测前准备

### 1. 启动 API 服务

```bash
# 进入 API 目录
cd apps/api

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev

# 或启动生产服务器
pnpm start
```

### 2. 验证服务状态

```bash
# 检查 API 健康状态
curl http://localhost:4000/health

# 预期响应
{"status":"ok","timestamp":"2026-04-02T00:00:00.000Z","version":"1.0.0"}
```

### 3. 准备测试数据

```bash
# 进入压测数据准备目录
cd apps/api/load-test

# 安装依赖
npm install

# 生成测试数据
node prepare-data.js
```

这会生成：
- 30 个测试账号（admin/creator/seeker 各 10 个）
- 100 个测试智能体
- 200 个测试帖子
- 测试 API Keys

### 4. 生成 JWT Tokens

```bash
# 运行准备脚本
cd apps/api/load-test
node prepare-data.js --tokens
```

脚本会输出各角色的测试 tokens，记录下来用于压测。

## 压测场景

### 场景 1: 首页 + 智能体列表（500 并发）

测试公开接口的高并发性能。

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --tags scenario=home_agents \
  --skip-scenario-errors
```

**测试接口：**
- GET /api/agents
- GET /api/agents/:id
- GET /api/community/posts

### 场景 2: AI 对话（50 并发）

测试 AI 接口的吞吐量和响应时间。

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --env AI_PROVIDER=zhipu \
  --env API_KEY=your_test_api_key \
  --tags scenario=ai_chat \
  --skip-scenario-errors
```

**测试接口：**
- POST /api/ai/chat
- POST /api/ai/chat/stream

### 场景 3: 登录防刷测试

验证登录接口的安全防护。

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --tags scenario=login_rapid \
  --skip-scenario-errors
```

**预期结果：**
- Rate Limit 生效后返回 429 状态码
- 5 次失败后账户被锁定

### 场景 4: 支付接口（100 并发）

测试支付相关接口。

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --tags scenario=payment \
  --skip-scenario-errors
```

**测试接口：**
- POST /api/payments/alipay
- POST /api/payments/wechat

### 场景 5: 文件上传接口

测试大文件上传性能。

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --tags scenario=upload \
  --skip-scenario-errors
```

**测试接口：**
- POST /api/upload

### 场景 6: 数据库高并发写入

测试评论、点赞等写入操作。

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --tags scenario=write_heavy \
  --skip-scenario-errors
```

**测试接口：**
- POST /api/community/comments
- POST /api/community/posts/:id/like

## 运行完整压测

### 方式 1: 运行所有场景

```bash
k6 run scenarios.js --env BASE_URL=http://localhost:4000
```

### 方式 2: 指定并发数

```bash
k6 run scenarios.js \
  --env BASE_URL=http://localhost:4000 \
  --duration 5m \
  --vus 100
```

### 方式 3: 使用自定义配置

编辑 `apps/api/load-test/config.js`：

```javascript
export const options = {
  scenarios: {
    home_and_agents: {
      executor: 'constant-vus',
      vus: 500,  // 并发数
      duration: '5m',  // 持续时间
    },
  },
}
```

## 压测后分析

### 性能瓶颈识别

1. **高延迟**
   - 检查数据库查询时间
   - 查看是否有 N+1 查询
   - 检查网络延迟

2. **高错误率**
   - 检查日志中的错误信息
   - 确认是否触发 Rate Limit
   - 检查服务是否过载

3. **高 CPU/内存**
   - 分析热点代码
   - 检查内存泄漏
   - 考虑扩容

### 优化建议

1. **数据库优化**
   - 添加适当索引
   - 使用连接池
   - 考虑读写分离

2. **缓存策略**
   - Redis 热点数据缓存
   - CDN 静态资源缓存
   - API 响应缓存

3. **扩展策略**
   - 水平扩展 API 实例
   - 使用负载均衡
   - 数据库分库分表
