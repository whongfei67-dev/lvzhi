## 前提条件

- [ ] 阿里云ECS实例已购买（推荐 Ubuntu 22.04 LTS）
- [ ] SSH密钥已配置（已有 `~/.ssh/aliyun_server`）
- [ ] 域名已购买并配置DNS解析
- [ ] 安全组已配置（开放 22, 80, 443 端口）

### 安全组配置

在阿里云控制台配置以下入方向规则：

| 协议 | 端口 | 授权对象 | 说明 |
|------|------|----------|------|
| TCP | 22 | 您的IP | SSH远程连接 |
| TCP | 80 | 0.0.0.0/0 | HTTP访问 |
| TCP | 443 | 0.0.0.0/0 | HTTPS访问 |

## 步骤1：连接服务器并检查环境

### 1.1 添加SSH配置（只需配置一次）

```bash
# 在本地 ~/.ssh/config 中添加（如果没有可以创建）
cat >> ~/.ssh/config << 'EOF'

# 律植服务器
Host aliyun
    HostName 8.159.156.192
    User root
    IdentityFile ~/.ssh/aliyun_server
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null
EOF

# 设置权限
chmod 600 ~/.ssh/config
```

### 1.2 连接服务器

```bash
ssh aliyun
```

### 1.3 检查服务器环境

连接成功后，运行以下命令检查：

```bash
echo "=== 系统信息 ==="
uname -a
cat /etc/os-release

echo ""
echo "=== 内存 ==="
free -h

echo ""
echo "=== 磁盘 ==="
df -h

echo ""
echo "=== Docker版本 ==="
docker --version

echo ""
echo "=== Docker Compose版本 ==="
docker-compose --version 2>/dev/null || docker compose version

echo ""
echo "=== Git版本 ==="
git --version
```

## 步骤2：安装Docker环境

### 2.1 如果Docker未安装，执行安装

```bash
# 更新系统
apt-get update && apt-get upgrade -y

# 安装依赖
apt-get install -y ca-certificates curl gnupg lsb-release

# 添加Docker GPG密钥
mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 添加Docker仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动Docker
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
docker compose version
```

### 2.2 配置Docker开机自启

```bash
systemctl enable docker
systemctl enable containerd
```

## 步骤4：配置环境变量

### 4.1 复制环境变量模板

```bash
cd /opt/lvzhi
cp deploy/env.example deploy/.env
```

### 4.2 编辑环境变量文件

```bash
nano /opt/lvzhi/deploy/.env
```

### 4.3 必需配置项

```bash
# ============================================
# 应用基础配置
# ============================================
NODE_ENV=production
APP_NAME=律植
APP_URL=https://www.lvzhi.cn  # 修改为您的域名
API_URL=https://www.lvzhi.cn/api  # 修改为您的域名

# ============================================
# 数据库配置 - 使用阿里云PolarDB
# ============================================
# 从阿里云控制台获取PolarDB连接信息
POSTGRES_HOST=your-pg-host.pg.rds.aliyuncs.com
POSTGRES_PORT=5432
POSTGRES_DB=lvzhi
POSTGRES_USER=lvzhi
POSTGRES_PASSWORD=your_secure_password
DATABASE_URL=postgresql://lvzhi:your_secure_password@your-pg-host.pg.rds.aliyuncs.com:5432/lvzhi

# ============================================
# Redis配置 - 使用阿里云Redis
# ============================================
REDIS_HOST=your-redis-host.redis.rds.aliyuncs.com
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

# ============================================
# AI服务配置
# ============================================
AI_PROVIDER=zhipu  # 或 minimax, dashscope 等
ZHIPU_API_KEY=your_zhipu_api_key

# ============================================
# JWT配置
# ============================================
JWT_SECRET=your_very_long_and_secure_jwt_secret_here_min_32_chars
JWT_EXPIRES_IN=7d
REFRESH_TOKEN_EXPIRES_IN=30d

# ============================================
# 安全配置
# ============================================
ALLOWED_ORIGINS=https://www.lvzhi.cn,https://lvzhi.cn  # 修改为您的域名
```

### 4.4 保存并退出

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

## 后续配置

### 域名DNS解析

在您的域名服务商处添加DNS记录：

| 记录类型 | 主机记录 | 记录值 |
|----------|----------|--------|
| A | @ | 8.159.156.192 |
| A | www | 8.159.156.192 |

### 配置SSL证书（推荐Let's Encrypt）

```bash
cd /opt/lvzhi

# 安装certbot
apt-get install -y certbot python3-certbot-nginx

# 申请证书（需要域名已解析）
certbot --nginx -d your-domain.cn -d www.your-domain.cn

# 自动续期
certbot renew --dry-run
```

### 配置防火墙

```bash
# 检查防火墙状态
ufw status

# 如果未启用，启用并配置
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable
```

### 设置自动重启

Docker Compose配置已包含 `restart: unless-stopped`，服务会在服务器重启后自动启动。

## 故障排查

### 服务无法启动

```bash
# 查看详细日志
docker compose logs -f

# 检查端口占用
netstat -tlnp | grep -E '80|443|3000|3001'
```

### 数据库连接失败

1. 检查PolarDB是否已开通公网地址
2. 确认安全组已开放5432端口
3. 验证DATABASE_URL配置正确

### 前端无法访问

1. 检查Nginx日志：`docker compose logs nginx`
2. 确认web容器状态：`docker compose ps web`
3. 检查Nginx配置：`cat /opt/lvzhi/deploy/nginx.conf`
