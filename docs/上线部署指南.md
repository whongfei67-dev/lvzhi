## 部署前检查清单

在开始部署前，请确保以下任务已完成：

### 基础设施配置

- [ ] 阿里云 ECS 实例已购买并运行
- [ ] 阿里云 PolarDB 已开通（华东2上海）
- [ ] 阿里云 OSS Bucket 已创建（lvzhi-assets）
- [ ] Docker 和 Docker Compose 已安装
- [ ] 域名已完成 ICP 备案

### 环境变量配置

在 `deploy/.env` 中配置以下环境变量：

```bash
# ============================================
# 数据库 - PolarDB
# ============================================
POSTGRES_HOST=pgm-xxxxxxxxxxxx.pg.rds.aliyuncs.com
POSTGRES_PORT=5432
POSTGRES_DB=lvzhi
POSTGRES_USER=lvzhi
POSTGRES_PASSWORD=你的密码
DATABASE_URL=postgresql://lvzhi:你的密码@pgm-xxxxxxxxxxxx.pg.rds.aliyuncs.com:5432/lvzhi

# ============================================
# Redis
# ============================================
REDIS_HOST=你的Redis地址
REDIS_PORT=6379
REDIS_PASSWORD=你的Redis密码

# ============================================
# OSS
# ============================================
ALIYUN_OSS_ENDPOINT=oss-cn-shanghai.aliyuncs.com
ALIYUN_OSS_BUCKET=lvzhi-assets
ALIYUN_OSS_ACCESS_KEY_ID=你的AccessKeyID
ALIYUN_OSS_ACCESS_KEY_SECRET=你的AccessKeySecret

# ============================================
# JWT
# ============================================
JWT_SECRET=生成一个32位以上的随机字符串
JWT_EXPIRES_IN=7d
REFRESH_TOKEN_EXPIRES_IN=30d

# ============================================
# AI 服务
# ============================================
ZHIPU_API_KEY=你的智谱AI密钥
DASHSCOPE_API_KEY=你的阿里云密钥
WENXIN_API_KEY=你的百度密钥

# ============================================
# 支付
# ============================================
ALIPAY_PID=2088xxxxxx
ALIPAY_APP_ID=20xxxxxx
ALIPAY_PRIVATE_KEY=...
ALIPAY_PUBLIC_KEY=...

WECHAT_PAY_MCH_ID=1xxxxxxx
WECHAT_PAY_API_KEY=...
```

### 生成 JWT Secret

```bash
# 使用 openssl 生成随机密钥
openssl rand -base64 32
```

## 部署步骤

### 步骤 1: 服务器准备

```bash
# 登录服务器
ssh root@你的服务器IP

# 创建应用目录
mkdir -p /opt/lvzhi
cd /opt/lvzhi

# 克隆代码（或使用 rsync 上传）
git clone https://github.com/whongfei67-dev/lvzhi-web.git .

# 安装 Docker（如果尚未安装）
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### 步骤 2: 配置环境变量

```bash
cd /opt/lvzhi

# 复制环境变量模板
cp deploy/env.example .env

# 编辑环境变量文件
nano .env
# 填入所有必要的配置
```

### 步骤 3: 数据库迁移

```bash
# 等待数据库连接
sleep 10

# 运行数据库迁移
cd /opt/lvzhi/apps/api
npm run db:migrate

# 验证迁移成功
npm run db:status
```

### 步骤 4: 构建并启动服务

```bash
cd /opt/lvzhi

# 构建 Docker 镜像
docker compose build

# 启动服务
docker compose up -d

# 查看服务状态
docker compose ps
```

### 步骤 5: 配置 Nginx

```bash
# 复制 Nginx 配置
cp /opt/lvzhi/deploy/nginx.conf /etc/nginx/conf.d/lvzhi.conf

# 测试配置
nginx -t

# 重新加载 Nginx
nginx -s reload
```

### 步骤 6: 配置 SSL 证书

```bash
# 创建证书目录
mkdir -p /etc/nginx/ssl

# 上传证书文件
# 将证书文件上传到 /etc/nginx/ssl/
# lvzhi.com.pem
# lvzhi.com.key

# 更新 Nginx 配置启用 HTTPS
nano /etc/nginx/conf.d/lvzhi.conf
```

### 步骤 7: 配置 DNS

在域名服务商处配置 DNS 解析：

```
A 记录：lvzhi.com → 服务器IP
A 记录：www.lvzhi.com → 服务器IP
CNAME：cdn.lvzhi.com → xxx.cdn.alibaba.com
```

## 灰度发布

### 灰度策略

采用渐进式流量切换策略：

| 阶段 | 流量比例 | 持续时间 | 验证内容 |
|------|----------|----------|----------|
| 1 | 5% | 30 分钟 | 核心功能可用 |
| 2 | 25% | 1 小时 | 性能无明显下降 |
| 3 | 50% | 2 小时 | 无重大错误 |
| 4 | 100% | - | 完全切换 |

### 执行灰度发布

```bash
# 启动灰度发布
cd /opt/lvzhi
node scripts/canary-deploy.js --step=5 --health-check

# 监控状态
node scripts/health-check.js --watch

# 推进到下一阶段
node scripts/canary-deploy.js --step=25 --health-check

# 完全切换
node scripts/canary-deploy.js --step=100
```

### 回滚操作

```bash
# 立即回滚
node scripts/canary-deploy.js --rollback

# 或切换到 Supabase
node scripts/migrate-users.js --rollback
```

## 常见问题处理

### 服务启动失败

```bash
# 查看容器日志
docker compose logs -f api
docker compose logs -f web

# 检查端口占用
netstat -tlnp | grep 3000
netstat -tlnp | grep 3001
```

### 数据库连接失败

```bash
# 测试数据库连接
psql "postgresql://lvzhi:密码@pgm-xxx.pg.rds.aliyuncs.com:5432/lvzhi"

# 检查 IP 白名单
# 阿里云 PolarDB 控制台 → 数据安全性 → 白名单设置
```

### Nginx 502 错误

```bash
# 检查后端服务状态
docker compose ps

# 检查 Nginx 日志
tail -f /var/log/nginx/error.log

# 重启服务
docker compose restart
```
