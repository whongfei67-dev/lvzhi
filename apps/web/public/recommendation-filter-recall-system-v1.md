# 律植筛选-召回算法（系统文字版 v2）

## 1. 目标

这套算法的目标是：即使用户表达不完整、关键词断裂、语序混乱，也能先识别最可能的案由，再给出既有精准度又有覆盖度的推荐结果。

最终推荐总量固定为：

- 帖子 5
- 技能 5
- 律师 5

---

## 2. 整体流程（从输入到推荐）

### 第一步：收集输入线索

- 合并用户第一句输入 + 固定问题回答；
- 抽取关键词、城市、需求方向、心智特征；
- 先做合法关键词标准化，避免虚构标签。

### 第二步：案由级联筛选（E -> D -> C -> B -> A）

不要求用户表达刚好落在 A/B/C/D/E 某个模板里，而是统一走级联收敛：

1. **E 层先筛**：用最小关键词下限做第一轮候选过滤，保留最可能的一批案由；
2. **D 层再筛**：处理碎片化表达（关键词分散、前后不连）；
3. **C 层再筛**：处理口语化表达（有主语、有目标，但信息不全）；
4. **B 层再筛**：用较完整的二级/三级语义继续收敛；
5. **A 层终筛**：用完整语义做最后判定；
6. 若仍有并列候选，用综合加权选出最终三级案由。

补充策略：

- 如果输入没有足够案由信号，不强行判定三级案由，回退到通用领域识别（劳动、合同、数据、知产等）。

### 第三步：按 L1/L2/L3 分层累积召回

确定案由后，不是“只取某一层”，而是按你定义的规则累积：

- **命中一级案由（L1）**：律师 +2
- **命中二级案由（L2）**：帖子 +2、技能 +2、律师 +2
- **命中三级案由（L3）**：帖子 +3、技能 +3、律师 +1

因此，当完整命中到三级案由时，合并结果是：

- 帖子 5（L2 的 2 + L3 的 3）
- 技能 5（L2 的 2 + L3 的 3）
- 律师 5（L1 的 2 + L2 的 2 + L3 的 1）

### 第四步：分层补位逻辑

如果某一层库存不足，不会直接断档，而是按优先顺序补齐：

- 帖子/技能：优先三级，再二级，再一级，再全局兜底
- 律师：优先同层案由，再城市优先（同城 > 近城），再作者优先和领域相关性排序

---

## 3. 推荐理由（自然语言版）

推荐理由不暴露算法分数，只给用户可理解的解释，核心围绕三件事：

1. **为什么是这个案由方向**
  - 例如：你提到的“支付令/申请”线索更符合督促程序相关案由，所以优先按这个方向检索。
2. **为什么先看到这些帖子和技能**
  - 例如：先展示与当前案由最接近的经验帖和实操技能，帮助你先判断路径和证据准备顺序。
3. **为什么是这几位律师**
  - 例如：优先同案由标签，再优先同城或近城，再考虑是否为相关内容创作者，保证可落地沟通效率。

可直接用于前端展示的自然语言示例：

- “根据您提供的线索，当前优先按该案由方向为您匹配内容，先给您可直接参考的帖子和实操技能，再匹配可沟通的律师。”
- “已优先展示与您问题最接近的案由内容；若同层内容不足，会补充同领域高相关内容，确保建议完整可执行。”
- “律师推荐会先看案由匹配，再优先同城/近城，方便您后续沟通与推进。”
