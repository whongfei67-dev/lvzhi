律植首页需求推荐：方案梳理（按你反馈修正版）

目标：搜索框作为核心分发入口，用户输入需求后，系统拆解关键词并在首页不同模块调用匹配内容。
重点：不把“推荐逻辑文案”硬写在页面显性位置，而是通过模块内容重排与调用结果体现智能分发。

一、你的新要求（本次增量）

  • 为提高拆解准确率：按案件类型增加固定追问树（例如劳动先问“劳动者/用人单位”再分支提问）。
  • 搜索框做成两行大输入框，问题在框内逐步引导，不额外弹窗。
  • 每轮问答后，问题与答案不保留在输入框，仅在后端/状态层做关键词留存。
  • 连续问 3 个问题，结合首次需求拆解词，形成总计 5-6 个关键词。
  • 精选技能模块逻辑：从全部技能中召回并展示匹配度最高的结果。
  • 统一调用规则：点击“生成推荐”后，基于留存的全部关键词统一召回。
  • 各模块展示数量固定：Top N = 3。
  • 问题解决型路由不变：社区帖子 → 技能 → 律师。
  • 合作连接型要拆独立路由：技能 → 合作机会 → 律师，不走社区入口。
  • 律师库补充规则：在“其他候选律师”阶段，优先选择地域 + 领域双匹配。

二、结构原则（保持不变）

  • 首页主结构不动，搜索框变大并提高视觉权重。
  • 搜索框支持两行：无输入态“第一行演示问题 + 第二行空白输入区”。
  • 输入后第一行切换为当前问题，第二行填写答案。
  • 按钮、卡片全部复用现有 UI 体系，不新增突兀样式。
  • 推荐逻辑不靠大文案堆叠，以“模块内容被替换和重排”体现。
  • 保留两行小字说明，删除示例输入标签和“下一步行动”卡片。
  • 首页不展示诉讼沙盘入口（保持独立工具边界）。

三、固定追问树（关键词增强）

以劳动争议为例（第一版）

  • Q1：你是劳动者还是用人单位？
  • Q2（劳动者分支）：是否有书面解除通知？是否签离职协议？
  • Q2（用人单位分支）：是否有绩效制度？是否有考核与改进记录？
  • Q3：当前证据有哪些？主要诉求是什么？
  • 每问一轮后：当前问答写入“关键词留存池”，输入框自动跳到下一个问题。
  • 输出：角色关键词 + 程序关键词 + 证据关键词 + 诉求关键词（用于召回排序）。
结论：自然语言输入 + 三轮结构化追问结果合并后再算匹配度，提升召回精度。

四、完整推荐链路（新）

Step 1 输入需求

用户输入自由文本。

→

Step 2 类型识别

识别为劳动/合同/知产等案件或合作连接诉求。

→

Step 3 固定追问

在搜索框内连续问 3 题，答案逐题留存。

→

Step 4 关键词融合

首次拆解 2-3 词 + 追问词，合成 5-6 个关键词包。

→

Step 5 双路分发

点击生成推荐后，统一按关键词包召回；各模块取 Top N=3。

首页仍不跳转：点击生成推荐后，直接刷新首页四模块内容与排序。

五、双路优先级（核心改版）

A. 找律师路子（问题解决）

  • 排序：社区帖子 → 技能 → 律师
  • 目标：先理解问题，再用工具，再找律师复核。

B. 找合作路子（资源连接）

  • 排序：技能 → 合作机会 → 律师
  • 说明：该路子不走社区帖子入口。

六、律师召回规则（修正版）

  • 模块说明小字：根据你的需求推荐可能匹配的律师。
  • 候选优先级：
    • ① 匹配关键词的技能作者（且为认证律师）
    • ② 匹配关键词的帖子发布者（且为认证律师）
    • ③ 其他律师库候选（地域 + 领域优先）
  • 过滤：非认证律师全部跳过，继续取下一个候选。
  • 卡片结构不变，仅增加推荐原因与匹配标签。

七、模块调用逻辑（按匹配度）

  • 技能模块：从全量技能池按关键词匹配分排序，取 Top N=3。
  • 帖子模块：仅在“找律师路子”参与召回，从社区帖子池按匹配度排序后取 Top N=3（即匹配度最高的帖子）。
  • 合作模块：仅在“找合作路子”高权重参与召回，从合作机会池取 Top N=3。
  • 律师模块：按“作者优先 + 认证过滤 + 地域领域兜底”取 Top N=3。

八、搜索框交互状态机（你指定的方式）

  • 状态 A（初始）：两行大搜索框；第一行显示演示问题，第二行空白待输入。
  • 状态 B（第 1 问）：第一行显示当前问题，第二行输入答案；提交后不回显历史问答。
  • 状态 C（第 2/3 问）：自动跳题，重复相同模式，持续写入关键词留存池。
  • 状态 D（可生成）：关键词达到 5-6 个后，点击“生成推荐”统一调用首页模块内容。
  • 状态 E（清除）：清空关键词池与推荐状态，恢复默认首页排序。

九、子页面联动排序（保持同一路由）

  • 进入子页后，使用相同关键词包与相同路由优先级做置顶。
  • 找律师路子：社区相关优先，其次技能，再律师。
  • 找合作路子：技能优先，其次合作，再律师。
  • 保留“清除推荐”后恢复各子页默认排序。

十、实施清单（确认后落地）

  • 新增“搜索框内三问引导”与关键词留存状态机（先做劳动争议）。
  • 改推荐引擎为“双路子分发策略”。
  • 改律师候选器：加地域 + 领域优先兜底。
  • 首页与子页同步按新优先级重排。
  • 统一模块 Top N=3 并固定推荐原因展示位。
